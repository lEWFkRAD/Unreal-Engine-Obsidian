# UE5 C++ Episode 1 - Project Setup and Getting Started

**Channel:** Code Tech | **Duration:** ~20:00

## Overview

This tutorial covers the fundamental workflow of using C++ in Unreal Engine — how to set up your project, how to launch it from your IDE, and the essential macros and file structure you need to understand.

---

## The Key Mindset Shift

Once you add C++ to your project, **you no longer launch from the Epic Games Launcher or Unreal Editor**. Instead, you launch your project from your code editor (Visual Studio, Rider, or CLion). The IDE builds the project and opens the editor automatically.

---

## Step 1: Create or Convert a Project

You can start with either a Blueprint or C++ project — it doesn't matter. You can always add C++ later.

**To add C++ to an existing Blueprint project:**
1. Go to **Tools → New C++ Class**
2. Choose a parent class (e.g., Actor)
3. Name your class (e.g., `MyActor`)
4. Click **Create Class**

The first time you do this, it takes several minutes as Unreal sets up the Visual Studio project files (`.sln`, `Build.cs`, etc.).

After creation, you'll see:
> "This project now includes sources. Close the editor and build from your IDE."

---

## Step 2: Launch from Visual Studio

1. Close the Unreal Editor
2. Open the `.sln` file in Visual Studio
3. Set the build configuration to **Development Editor** (Win64)
4. Press **F5** or the **Play** button to build and launch

Visual Studio will compile the project and open the Unreal Editor with your project loaded. This is now your standard workflow.

---

## Step 3: Understanding the Generated Files

When you create a C++ class, Unreal generates two files:

### Header File (.h) — Class Definition

```cpp
// MyActor.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "MyActor.generated.h"

UCLASS()
class YOURPROJECT_API AMyActor : public AActor
{
    GENERATED_BODY()

public:
    AMyActor();

protected:
    virtual void BeginPlay() override;

public:
    virtual void Tick(float DeltaTime) override;
};
```

### Source File (.cpp) — Implementation

```cpp
// MyActor.cpp
#include "MyActor.h"

AMyActor::AMyActor()
{
    PrimaryActorTick.bCanEverTick = true;
}

void AMyActor::BeginPlay()
{
    Super::BeginPlay();
}

void AMyActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
}
```

## Step 4: Essential Macros Explained

### `#include "MyActor.generated.h"`
**Required in every Unreal header file.** This is auto-generated by Unreal Header Tool (UHT) and enables the reflection system — allowing the editor to see your classes, variables, and functions.

### `UCLASS()`
Marks a class as an Unreal Engine class. Required for any class that inherits from `UObject` (including `AActor`). Must be placed directly above the class declaration.

### `GENERATED_BODY()`
**Required inside every UCLASS.** Generates boilerplate code for the reflection system. Place it at the top of your class body.

### `UPROPERTY()`
Marks a variable for Unreal's reflection system. Without it, the editor cannot see or interact with your variable.

```cpp
// Basic — editor can see but not edit
UPROPERTY()
float Health;

// Editable in details panel, readable/writable from blueprints
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
float MaxHealth;

// Visible but not editable (great for debugging)
UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Stats")
float CurrentHealth;
```

### `UFUNCTION()`
Marks a function for Unreal's reflection system.

```cpp
// Callable from blueprints
UFUNCTION(BlueprintCallable, Category = "Actions")
void Attack();

// Pure function — no execution pin in blueprints (good for getters)
UFUNCTION(BlueprintPure, Category = "Stats")
float GetHealthPercent() const;

// Can be overridden in blueprints
UFUNCTION(BlueprintNativeEvent, Category = "Actions")
void OnDeath();

// Must be implemented in blueprints (no C++ body needed)
UFUNCTION(BlueprintImplementableEvent, Category = "Actions")
void OnLevelUp();
```

## Step 5: Adding Variables to Your Actor

```cpp
// MyActor.h — in the class body

// Private variable (not visible in editor by default)
int32 TestValue;

// Exposed to editor and blueprints
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Config")
float Speed = 100.0f;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Config")
int32 Score;

UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Debug")
FString ActorName;
```

**Important:** If a variable is in the `private` section and you want to expose it:

```cpp
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Config",
    meta = (AllowPrivateAccess = "true"))
float PrivateSpeed;
```

## Step 6: Recompiling Workflow

After making code changes:

1. **Full rebuild**: Close editor → build from Visual Studio → relaunch
2. **Live Coding** (hot reload): Press `Ctrl+Alt+F11` in the editor, or click the Live Coding button in the toolbar

**Note:** Live Coding has limitations — changes to `UCLASS`, `UPROPERTY`, or `UFUNCTION` declarations sometimes require a full rebuild.

---

## Step 7: Creating a Blank Level for Testing

The default level includes tons of content. For clean testing:

1. Content Browser → Right-click → **Level**
2. Name it `BlankLevel`
3. Open it — it's truly empty

---

## Key Workflow Summary

```
1. Write code in Visual Studio / Rider
2. Build (Ctrl+Shift+B or F5)
3. Editor opens automatically
4. Test in editor
5. Make changes → Live Coding (Ctrl+Alt+F11) or full rebuild
6. Repeat
```

## Naming Conventions

| Type | Prefix | Example |
|------|--------|---------|
| Actor | `A` | `AMyActor` |
| UObject | `U` | `UMyComponent` |
| Struct | `F` | `FMyStruct` |
| Interface | `I` | `IMyInterface` |
| Enum | `E` | `EMyEnum` |
| Boolean | `b` | `bIsAlive` |

These prefixes are not just conventions — Unreal's code generation depends on them.
